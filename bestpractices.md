The goal is simple: produce work that a busy person can use.

### The standard (what “good” looks like)

A strong piece is:

- **Clear**: the main claim is obvious in the first 3–5 sentences  
- **Grounded**: sources are included and claims match the evidence  
- **Actionable**: it points to a decision, recommendation, or next step  
- **Skimmable**: headings, short paragraphs, and a clean structure  
- **Honest**: names uncertainty and tradeoffs instead of hiding them  

### Define done first

Before you start, write:

**Done Definition:**  
“One [type of piece] for [audience] that answers [question] and ends with [decision/recommendation].”

Examples:
- “A 1–2 page brief for city staff that explains X and recommends one action.”
- “A chart + 200-word caption that shows Y over time and states the takeaway.”
- “A memo that compares 2–3 options and recommends one.”

If you can’t write this sentence, tighten scope before writing.

### Use one of these structures

**Brief**  
Problem → What’s true now → Options → Recommendation → Sources

**Essay**  
Thesis → Why it matters → Evidence → Strongest objection → Conclusion

**Memo**  
Background → Decision needed → Analysis → Next steps

### Make the current situation legible

Include the few variables that actually explain what’s going on:
- constraints (money, law, capacity, time)
- incentives (who benefits, who pays)
- key numbers (the 3–5 facts that define the situation)
- assumptions (what you’re taking as true)

### Persuasion without nonsense

If you want people to care, give them something they can picture:
- a concrete example
- a before/after scenario
- a short “what this means for normal people” paragraph

No melodrama. No vague moralizing. Evidence first.

### Feedback and revision

Ask for feedback early using these prompts:
- “Is my main claim obvious?”
- “Where does it get confusing?”
- “What’s the strongest counterargument?”
- “What’s one thing I should cut?”

Then revise once with focus: fix the biggest weakness first.

### Final check before publishing

- Does the first paragraph say what this is and why it matters?
- Are claims supported with links or citations?
- Are tradeoffs or uncertainty acknowledged?
- Can someone scan headings and get the story?
- Is the conclusion a decision, recommendation, or next step?
